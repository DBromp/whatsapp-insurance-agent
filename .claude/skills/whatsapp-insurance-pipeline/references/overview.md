# Project brief

Source: `Teste Técnico de Data & AI Engineering.pdf` and `Dicionário de Dados.pdf` (in the project folder).

## Objective

Build an AI agent that creates, manages, and maintains a 3-layer medallion data pipeline over WhatsApp sales conversations between auto-insurance leads and human sellers.

## Mandatory requirements

1. Pure Python pipeline
2. AI Agent that creates AND manages the pipeline (with self-correction on failure)
3. Live Gold layer — auto-update when source data grows
4. Three clear medallion layers with explicit transformations
5. Public GitHub repo

## Differentiators (bonus credit)

1. Use of Databricks (free tier is fine) — explicitly highlighted
2. Creative classifications and insights in Gold
3. Strong cleaning and transformation approach
4. Creative segmentations
5. Robust self-correction mechanism
6. Going beyond the example insights (which are discouraged)

## Evaluation axes

- Data engineering + analytical creativity: robust layered pipeline, infra vs. ad-hoc analysis, insight quality
- Agent capability: autonomy, self-correction, continuous pipeline management
- Code quality: clean, structured, documented, best practices

## Scope and timeline

4-day full delivery. See the parent project session for the day-by-day plan: Day 1 = foundation, Day 2 = Silver, Day 3 = Gold + incremental, Day 4 = agent + polish.
